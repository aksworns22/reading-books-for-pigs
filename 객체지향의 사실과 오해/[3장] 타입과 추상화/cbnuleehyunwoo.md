- **어떤 추상화도 의도된 목적이 아닌 다른 목적으로 사용된다면 오도(그릇된 길로 이끌게)될 수 있다!**
- 이 책에서는 추상화를 다음과 같이 정의한다.

    >     - 두 번째 차원은 중요한 부분을 강조하기 위해 불필요한 세부사항을 제거함으로써 단순하게 만드는 것이다.
    > - 모든 경우에 추상화의 목적은 복잡성을 이해하기 쉬운 수준으로 단순화 하는 것이다.

## 객체지향과 추상화

### 모두 트럼프일 뿐,

> …… 마침내 하트 왕과 하트 여왕이 그 모습을 드러냈다.

앨리스는 여왕을 쳐다보며 마음속으로 이렇게 속삭였다.

”기껏해야 트럼프에 불과해, 무서워할 필요 없어”
> 
- 해당 장면에는 수많은 객체들이 등장한다(앨리스, 정원사, 클로버 병사, 신하, 공주, 왕자, 왕, 왕비 등등…)
- 위 객체들은 각각 다른 모습을 하고 있지만, 앨리스는 과감하게 (원래 다들 본질은 트럼프 카드이므로) 다양한 객체들을 추상화 하여 바라본 것.

### 개념

- 공통점을 기반으로 객체들을 묶기 위한 그릇!
- 우리는 사실 주변에 존재하는 다양한 사물들에게 특별한 개념을 적용하는데에 익숙하다.
    - ex>
        - 길거리를 빠르게 누비는 교통수단 → 자동차
        - 하늘을 나는 교통수단 → 비행기
        - 등등…
- 개념을 이용하면 객체를 여러 그룹으로 분류할 수 있다!
- 객체는 어떤 개념의 인스턴스이다.
    - 하얀 토끼는 토끼 개념 그룹의 일원(인스턴스)이고
    - 하트 여왕은 트럼프 개념 그룹의 일원이다.
- 따라서 객체는 다음과 같이 정의할 수 있다.
    
    > 특정한 개념을 적용할 수 있는 구체적인 사물을 의미한다. 개념이 객체에 적용되었을 때 객체를 개념의 인스턴스라고 한다.
    > 

### 개념의 세 가지 관점

- 일반적으로 객체의 분류 장치로서 개념을 이야기 할 때는 아래의 세 가지 관점을 함께 언급한다.
    - 심볼: 개념을 가리키는 간략한 이름이나 명칭
    - 내연(intension): 개념의 완전한 정의를 나타내며 내연의 의미를 이용해 객체가 개념에 속하는지 여부를 확인 가능
    - 외연(extension): 개념에 속하는 모든 객체의 집합이다.
- 이미 언급되었던 내용으로 설명 가능하다고 한다(~~예?~~)

> 여러분은 앨리스가 정원사, 병사, 신하, 왕자와 공부, 하객으로 참석한 왕과 왕비들… 의 몸이 납작하고 두 손과 두 발이 네모 귀퉁이에 달려있기 때문에 트럼프라는 개념으로 묶었다는 사실을 잘 알고 있다
> 
- 위의 문장에 심볼, 내연, 외연이 모두 포함되어있다!
    - 심볼 → 트럼프
    - 내연: 개념을 객체에게 적용할 수 있는가? 를 판단하는 조건(~~하고 ~~하기 때문에 트럼프라는 개념으로 묶었다)
    - 외연: 개념에 속하는 객체들 즉 인스턴스들이 모여 이뤄진 집합을 가리킴

### 객체를 분류하기 위한 틀

- 동일한 개념으로 구성된 객체 집합에 해당 객체를 포함시킨다는 것은 무엇인가?
    - 어떤 객체에 어떤 개념을 적용할 수 있다!
- 어떤 객체를 어떤 개념으로 분류할지가 객체지향의 품질을 결정한다.
- 객체는 소중하다.. 따라서 객체를 안전하고 적절한 장소에 보관할 수 있도록 최대한 직관적으로 분류하라

### 분류는 추상화를 위한 도구다

- 개념은 추상화의 첫 번째 차원인 일반화를 적용한 결과이다.

    - 추상화를 사용함으로써 극도로 복잡한 세상을 제어 가능한 수준으로 단순화 하자…

## 타입

### 타입은 개념이다.

- 컴퓨터 공학자들은 MZ이다.
    - 개념이라는 단어는 우리의 일상에서 폭넓게 사용되는 단어이다,
    - 컴퓨터 공학자들은 개념을 좀 더 인텔리하게 표현할 수 있는 자신만의 용어를 가지고 싶어, 다른 용어를 수학에서 차용해왔다.
    - 타입은 개념이다!
        - 그렇지만 타입은 컴퓨터 세계로 들어온 순간 좀 더 기계적인 의미를 부여받게 되었다..

### 데이터 타입

- 컴퓨터가 어떤 작업을 수행하기 위해서는 작업에 필요한 데이터를 메모리 안으로 불러들여야 한다!
- 어떤 메모리 조각에 들어있는 값의 의미는 외부의 해석가에 의해 결정된다.
- 타입 시스템의 목적은 무엇인가?
    - 메모리 안의 모든 데이터가 비트열로 보임으로써 야기되는 **혼란 방지**
- 결론적으로 타입은….
    - 데이터가 어떻게 사용되는가?
        - ex> 숫자형 데이터이면 사칙연산에
        - 문자열 데이터이면 문자열 생성에…
    - 타입에 속한 데이터를 메모리에서 어떻게 표현하는지 외부로부터 감출 수 있다!
        - 해당 데이터 타입을 메모리에서 어떻게 표현하는지 몰라도 각 연산을 할 수 있다!
- 이 책에서는 프로그래밍 언어 관점에서 데이터 타입을 다음과 같이 정의한다.
    
    > 데이터 타입은 메모리 안에 저장된 데이터의 종류를 분류하는데 사용하는 메모리 집합에 관한 메타데이터이다. 데이터에 대한 분류는 암시적으로 어떤 종류의 연산이 해당 데이터에 대해 수행될 수 있는지를 결정한다.
    > 

### 객체와 타입

- 지금까지 왜 타입에 대해 설명했는가?
    - 전통적인 데이터 타입과 객체지향의 타입에는 깊은 연관성이 있기 때문이다.
- 마치 객체지향에서 객체를 타입에 따라 분류하고, 그 타입에 이름을 붙이는 것이 새로운 데이터 타입을 선언하는 것처럼 볼 수 있을 것이다.
- 하지만 객체 == 데이터는 절대 아님!
    - 객체에서 중요한 것은 객체의 행동
- 그렇다면 객체의 타입을 결정하는 것도 행동이어야 한다.
    1. 어떤 객체들이 동일한 행동을 수행할 수 있다면, 동일한 타입으로 분류될 수 있다. (첫 번째 조언)
    2. 객체의 내부적인 표현은 외부로부터 철저히 감춰진다. 즉 객체의 행동을 가장 효과적으로 수행할 수만 있다면 객체 내부의 상태를 어떤 방식으로 표현하더라도 무방하다. (두 번째 조언)

### 행동이 우선이다.

- 두 번째 조언에 따르면 객체의 타입은 객체의 내부 표현과는 아무런 상관이 없음,
    - → 객체의 내부 표현 방식이 다르더라도 어떤 객체들이 동일하게 행동한다면 그 객체들은 동일한 타입에 속한다.
- 동일한 행동 == 동일한 책임 == 동일한 메시지 수신
- 동일한 타입에 속한 객체는 내부의 데이터 표현 방식이 다르더라도 동일한 메시지를 표현하고 이를 처리할 수 있음
    - 내부적으로 동일한 메시지를 처리하는 방법은 다르겠지만..

## 타입의 계층

### 트럼프 계층

- 정원사, 병사, 신하, 왕자.. 등등은 정말로 트럼프인가?(카드 게임을 할때 사용하는 그 트럼프 카드)
    - 그렇지 않다. 단지 트럼프 카드와 몇몇가지 특징을 공유하기 때문에
    - 좀 더 엄밀히 말하면 트럼프 인간?으로 볼 수 있을 것
    - 즉 그냥 트럼프 카드가 할 수 있는 일을 트럼프 인간은 할 수 있으며, 심지어 걸어다니는 행동을 추가로 할 수 있다.
    - 즉 모든 트럼프 인간은 트럼프가 될 수 있지만, 모든 트럼프가 트럼프 인간이 될 수는 없다.

### 일반화/특수화 관계

- 일반적인 타입은 특수한 타입에 비해 더 적은 수의 행동을 가지며 특수한 타입은 일반적인 타입에 비해 더 많은 행동을 가진다
- 특수한 타입은 일반적인 타입이 할 수 있는 모든 행동을 동일하게 수행할 수 있어야 한다.

### 슈퍼타입과 서브타입

- 일반화/특수화 관계는 일반적인 타입/특수한 타입 간의 관계이다.
- 여기서 좀 더 일반적인 타입을 슈퍼타입이라고 하, 좀 더 특수한 타입을 서브타입이라고 한다.
- 일반적으로 서브타입은 슈퍼타입의 행위와 호환되기 때문에 서브타입은 슈퍼타입을 대체할 수 있어야 한다.

### 일반화는 추상화를 위한 도구다

> 기껏해야 트럼프에 불과해
> 
- 위 엘리스의 말은 두 가지 기법이 사용되었다.
    1. 정원에 있던 등장인물의 차이점은 배제하고 공통점만을 강조 → 공통의 타입인 트럼프 인간으로 분류
    2. 트럼프 인간을 좀 더 단순한 관점에서 바라보기 위해 불필요한 특성을 배제하고 좀 더 포괄적인 의미를 지닌 트럼프로 일반화 했다는 것 

## 정적 모델

### 타입과 목적

- 왜 타입을 사용해야 할까?
    - 인간의 인지 능력으로 시간에 따라 동적으로 변하는 객체의 복잡성을 극복하기가 너무 어렵기 때문에
- 타입은 시간에 따라 동적으로 변하는 엘리스의 상태를 시간과 무관한 정적인 모습으로 다룰 수 있게 해줌
    - 결국 타입은 시간이라는 요소를 제거함으로써, 시간에 독립적인 정적인 모습으로 엘리스를 생각할 수 있게 해준다.

### 동적 모델과 정적 모델

- 객체를 생각할 때 두 가지 모델을 동시에 고려한다.
    1. 객체가 특정 시점에 구체적으로 어떤 상태를 가지는가?(스냅샷)
        - 즉 객체가 살아 움직이는 동안 상태가 어떻게 변하고 어떻게 행동하는지를 표현
    2. 객체가 가질 수 있는 모든 상태와 행동을 시간에 독립적으로 표현하는 것
        - 동적으로 변하는 객체의 상태가 아니라 객체가 속한 타입의 정적인 모습을 표현하기 때문에 정적 모델이라고도 한다.
- 객체지향 애플리케이션을 설계하고 구현하기 위해서는 동적모델과 정적 모델을 적절히 혼용해야 한다!

### 클래스

- 클래스와 타입은 동일한 개념이 아니다.
- 객체를 분류하는 기준은 타입이며, 타입을 나누는 기준은 객체가 수행하는 행동이다…
- 클래스는 단지 타입을 결정한 후 프로그래밍 언어를 이용해 타입을 구현할 수 있는 한 가지 방법이 클래스라는 사실을 기억하자…
