# 협력하는 객체들의 공동체

객체지향의 세계를 설명할 때 실세계의 모방이라는 설명을 자주하는데 이건 좋은 비유는 아니다. 그럼에도 불구하고 실세계의 모방이라는 객체지향의 개념은 훌륭한 프로그램을 설계하고 규현하는 실무적인 관점에서는 부적합하지만, 객체지향이라는 용어에 담긴 기본 사상을 이해하고 학습하는 데 효과적이다.

## 협력하는 사람들

1. 손님, 캐시어, 바리스타는 주문한 커피를 손님에게 제공하기 위해 협력하는 과정에서 자신이 맡은 바 **책임**을 다한다.
2. 커피 주문이라는 **협력**에 참여하는 모든 사람들은 커피가 정확하게 주문되고 주문된 커피가 손님에게 정확하게 전달될 수 있도록 맡은 바 **역할**과 **책임**을 다한다.

여러 사람이 동일한 역할을 수행할 수 있다.

- 역할이라는 단어는 책임을 내포한다.
- 누가 주문을 받든, 누가 커피를 만들든 손님 입장에서는 상관하지 않는다.

역할은 대체 가능성을 의미한다.

책임을 수행하는 방법은 자율적으로 선택할 수 있다.

- 책임을 수행하기만 한다면 다른 방법을 취하더라도 상관 없다. 이를 다형성이라 한다.

한 사람이 동시에 여러 역할을 수행할 수 있다.

## 역할, 책임, 협력

객체, 메세지, 메서드

### 상태와 행동을 함께 지닌 자율적인 객체

객체는 다른 객체가 무엇을 수행하는지는 알 수 있지만, 어떻게 수행하는지는 몰라야 한다.

한 객체가 다른 객체에게 요청하는 것을 메세지를 전송한다고 칭한다.

- 요청을 받는 건 메세지를 수신 받는다고 한다.

객체가 수신된 메세지를 처리하는 방법을 메서드라 칭한다.

## 객체지향의 본질

1. 객체지향이란 시스템을 상호작용하는 자율적인 객체주의 공동체로 바라보고 객체를 이용해 시스템을 분할하는 방법
2. 자율적인 객체란, 상태의 행위를 함께 지니며 스스로 자기 자신을 책임지는 객체를 의미한다.
3. 객체는 시스템의 행위를 구현하기 위해 다른 객체와 협력한다.
4. 객체는 다른 객체와 협력하기 위해 메시지를 전송하고, 메시지를 수신한 객체는 메시지를 처리하는 데 적합한 메서드를 자율적으로 선택한다.
